# Sairo — web UI for MinIO

[Sairo](https://sairo.dev/) is a self-hosted S3 storage browser. On the `nixos` host it runs via **podman compose** and connects to the local MinIO API on port **9000**.

Access is available from the LAN (`enp2s0`, `wlp3s0`) and over **Tailscale** (`tailscale0` is trusted by the firewall).

## Open the UI

```
http://nixos:8000
```

Use the Tailscale hostname if needed (e.g. `http://nixos.tail1234.ts.net:8000`).

Default username is `admin`. The password is in `compose/sairo/.env` (from [`secrets/sairo.yaml`](../secrets/sairo.yaml)):

```bash
cd secrets
SOPS_AGE_KEY_FILE=age-keys.txt sops -d sairo.yaml
```

Look for `ADMIN_PASS` inside the `sairo-env` block.

## MinIO connection

Sairo reaches MinIO on the host at `http://host.containers.internal:9000` (set in `docker-compose.yml`). Podman injects this hostname into the container’s `/etc/hosts`. Do not use `127.0.0.1` or `host.docker.internal` — the former points at the container itself; the latter is unreliable with `podman compose` on Linux.

In **S3 Endpoints** (UI), set the URL to the address your **browser** uses for MinIO, e.g. `http://nixos:9000` or `http://192.168.8.157:9000` (path-style on). That origin must match `S3_PUBLIC_ORIGIN` in `.env` — nginx uses it in the Content-Security-Policy for image and PDF previews.

## MinIO service account

Sairo uses a dedicated MinIO service account (not root credentials). Create or rotate it with `mc`:

```bash
# Root credentials — from secrets/minio.yaml
cd secrets
SOPS_AGE_KEY_FILE=age-keys.txt sops -d minio.yaml

mc alias set local http://127.0.0.1:9000 <MINIO_ROOT_USER> <MINIO_ROOT_PASSWORD>
mc admin user svcacct add local <MINIO_ROOT_USER> --name sairo --json
```

Copy the `accessKey` and `secretKey` from the JSON output into `secrets/sairo.yaml` under the `sairo-env` key (dotenv format), then re-encrypt:

```yaml
sairo-env: |
    S3_ACCESS_KEY=<accessKey>
    S3_SECRET_KEY=<secretKey>
    ADMIN_PASS=<strong password>
    JWT_SECRET=<openssl rand -hex 32>
    S3_PUBLIC_ORIGIN=http://192.168.8.157:9000
```

```bash
cd secrets
SOPS_AGE_KEY_FILE=age-keys.txt sops sairo.yaml   # edit sairo-env, save — sops re-encrypts on exit
sudo nixos-rebuild switch --flake .#nixos
```

After rebuild, sops-nix writes `/etc/nixos/compose/sairo/.env` automatically.

To remove an old account before creating a new one:

```bash
mc admin user svcacct remove local <OLD_ACCESS_KEY>
```

## Troubleshooting

| Symptom | Likely cause |
|---------|----------------|
| `Could not connect` / blank UI | Container not running — check `systemctl status sairo-compose` |
| S3 errors after login | Wrong or expired keys in `secrets/sairo.yaml` / `compose/sairo/.env` |
| Login cookie not saved | `SECURE_COOKIE` must be `false` for plain HTTP (set in compose) |
| Slow first load | Sairo is indexing buckets into SQLite under `/var/lib/sairo/data` |
| `DB_DIR '/data' is not writable` | Host dir must be owned by UID 1000 — `sudo chown -R 1000:1000 /var/lib/sairo/data` |
| `Connect timeout` to `host.containers.internal:9000` (hosts file looks fine) | NixOS firewall blocks podman bridge → host; ensure `minio.nix` allows port 9000 on `podman+`, then `sudo nixos-rebuild switch` |
| `Connect timeout` on `host.docker.internal:9000` | In S3 Endpoints, set URL to `http://host.containers.internal:9000` (path-style on), or clear `/var/lib/sairo/data` and restart |
| `Could not connect to the endpoint URL` (127.0.0.1:9000) | Edit endpoint in UI to `http://host.containers.internal:9000`, or clear `/var/lib/sairo/data` and restart |
| CSP blocks `img-src` at `:9000` (preview broken, listing works) | Set `S3_PUBLIC_ORIGIN` in `secrets/sairo.yaml` to match the endpoint URL in Sairo UI (e.g. `http://192.168.8.157:9000`), rebuild, restart `sairo-compose` |

Check services:

```bash
systemctl status minio podman sairo-compose
journalctl -u sairo-compose -n 50
podman compose -f /etc/nixos/compose/sairo/docker-compose.yml ps
curl -I http://127.0.0.1:8000/healthz
curl -I http://127.0.0.1:9000/minio/health/live
```

## Sairo vs MinIO Console

| Service | Port | Purpose |
|---------|------|---------|
| Sairo | 8000 | Fast search and browse across buckets |
| MinIO Console | 9001 | Native MinIO admin UI |

Both access the same buckets and objects. Changes in either UI are visible in the other.
